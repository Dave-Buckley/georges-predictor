**Create `src/components/predictions/prediction-inputs.tsx`:**

A 'use client' component that renders two score stepper inputs (home and away) for a single fixture. This goes inside the prediction-area on each FixtureCard.

Props interface:
```typescript
interface PredictionInputsProps {
  fixtureId: string
  homeScore: number | null    // null = no prediction yet
  awayScore: number | null
  onChange: (fixtureId: string, home: number | null, away: number | null) => void
  disabled?: boolean           // true for locked/kicked-off fixtures
  hasSubmitted?: boolean       // true if this fixture has a saved prediction
}
```

**Stepper UX (locked decision):**
- Each score shows: [ - ] {score} [ + ] layout horizontally
- Score input is `<input type="number" inputMode="numeric" min={0} max={20}>` so tapping opens phone numeric keypad (fallback)
- +/- buttons are `<button type="button">` with Minus and Plus icons from lucide-react
- Minus button disabled when score is 0 or null; Plus disabled when score is 20
- When score is null (no prediction), show empty input with placeholder "—"
- First tap on + sets score to 0 then increments; or user can tap the number field to type directly
- Both inputs on one row: `Home [stepper] — [stepper] Away` label pattern
- When `disabled=true`, inputs are read-only: show the saved score values in a muted/greyed style (no stepper buttons)
- When `hasSubmitted=true` and `disabled=false`, show scores with a subtle green-tinted left border to indicate "saved"

**Visual distinction (Claude's discretion decisions):**
- Un-submitted fixture with no prediction: empty inputs, normal card
- Un-submitted fixture with typed scores (not yet saved): inputs with values, no green indicator
- Submitted fixture (saved to DB): green-tinted left border on the prediction area
- Locked fixture with prediction: greyed read-only scores shown
- Locked fixture without prediction: text "No prediction" in muted style

**Create `src/components/predictions/prediction-form.tsx`:**

A 'use client' component that wraps the entire gameweek fixture list with prediction form state management.

Props interface:
```typescript
interface PredictionFormProps {
  fixtures: FixtureWithTeams[]
  gameweek: GameweekRow
  existingPredictions: Record<string, { home_score: number; away_score: number }>
  submissionCount: { submitted: number; total: number }
  navGameweeks: Array<{ number: number; status: GameweekStatus }>
  currentGw: number
  totalGw: number
}
```

**State management:**
- Local state: `predictions: Record<string, { home_score: number | null; away_score: number | null }>` — initialized from existingPredictions prop (existing saved scores) merged with null entries for fixtures without predictions
- Local state: `isSubmitting: boolean` for loading state
- Local state: `submitResult: { saved: number; skipped: number } | null` for feedback
- Local state: `hasExistingPredictions: boolean` — derived from whether any existingPredictions were passed in (determines "Submit" vs "Update" button text)

**`handleScoreChange(fixtureId, home, away)`:** Updates local predictions state. No server calls (locked decision: no auto-save).

**`handleSubmit()`:**
1. Collect all predictions where BOTH home_score and away_score are not null (skip partially filled)
2. Filter out fixtures that are past kickoff (client-side pre-filter for UX; server still enforces)
3. If no valid entries, show inline message "No predictions to submit"
4. Call `submitPredictions(currentGw, entries)` server action
5. On success: show success banner with "Saved N predictions" (and "M skipped — already kicked off" if skipped > 0)
6. Update hasExistingPredictions to true so button shows "Update Predictions"
7. On error: show error banner with the error message

**Submission counter (locked decision):**
- Render above the fixture list: "X of Y members have submitted" in a subtle info bar
- Use Lucide `Users` icon + count text
- Styling: rounded bar, slate-700 background, white text, small font

**Late submission messaging (Claude's discretion):**
- If some fixtures are past kickoff, show a small amber banner above the submit button: "Some fixtures have kicked off — you can still predict the remaining ones"

**Confirmation feedback (Claude's discretion):**
- Use an inline success/error banner at the bottom of the page (above the submit button) that auto-dismisses after 5 seconds
- Green for success, red for error

**Layout:**
1. Submission counter bar
2. GameweekNav (passed through)
3. GameweekView (with predictions and onScoreChange passed in)
4. Late submission warning (conditional)
5. Feedback banner (conditional)
6. Submit/Update button (sticky at bottom for mobile: `sticky bottom-0 bg-slate-900 p-4 border-t border-slate-700`)

The submit button: full-width, large touch target (h-12), purple-600 background. Text: "Submit Predictions" or "Update Predictions". Disabled + spinner when isSubmitting.
