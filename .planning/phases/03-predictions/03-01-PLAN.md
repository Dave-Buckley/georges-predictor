**Function signature:**
```typescript
export async function submitPredictions(
  gameweekNumber: number,
  entries: Array<{ fixture_id: string; home_score: number; away_score: number }>
): Promise<{ success?: boolean; saved: number; skipped: number; error?: string }>
```

**Implementation flow:**
1. Create Supabase server client, get authenticated user via `getUser()` (not getSession — security pattern from Phase 1)
2. If no user, return `{ error: 'Not authenticated', saved: 0, skipped: 0 }`
3. Look up member row from `members` table where `user_id = user.id`, check `approval_status === 'approved'`
4. If not approved member, return `{ error: 'Member not approved', saved: 0, skipped: 0 }`
5. Validate input with `submitPredictionsSchema.safeParse({ gameweek_number: gameweekNumber, entries })`
6. If validation fails, return first error message
7. Loop through validated entries:
   - Call `canSubmitPrediction(entry.fixture_id)` — if `!lockout.canSubmit`, increment skipped, continue
   - Upsert to predictions table with `onConflict: 'member_id,fixture_id'` — set home_score, away_score, updated_at
   - If upsert succeeds, increment saved; if error, increment skipped
8. Call `revalidatePath('/gameweeks/' + gameweekNumber)`
9. Return `{ success: true, saved, skipped }`

**IMPORTANT:** Do NOT use `createAdminClient()` for this action. Member predictions use the member's own session client which respects RLS. The member_id is looked up server-side from auth.uid() — NEVER trust a client-passed member_id.
