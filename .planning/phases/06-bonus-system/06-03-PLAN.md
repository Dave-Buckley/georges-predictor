    3. After the prediction_scores upsert (Step 3), add Step 4 — bonus calculation:

       ```typescript
       // Step 4: Calculate bonus points for members who picked this fixture
       let bonus_calculated = 0

       const { data: bonusAwards, error: bonusQueryError } = await adminClient
         .from('bonus_awards')
         .select('id, member_id, bonus_type_id, awarded, bonus_type:bonus_types!bonus_type_id(name)')
         .eq('fixture_id', fixtureId)

       if (!bonusQueryError && bonusAwards && bonusAwards.length > 0) {
         for (const award of bonusAwards) {
           // Skip already-confirmed or already-rejected awards (don't overwrite George's decision)
           // We DO recalculate for pending awards (awarded IS NULL) — George hasn't reviewed yet
           const bonusTypeName = (award.bonus_type as any)?.name ?? ''

           // Find this member's prediction_scores for the same fixture
           const memberScore = scoreRows.find((s: any) => s.member_id === award.member_id)

           if (memberScore) {
             const bonusResult = calculateBonusPoints(
               bonusTypeName,
               { result_correct: memberScore.result_correct, score_correct: memberScore.score_correct },
               { home: homeScore, away: awayScore },
             )

             // Update the bonus_awards row with calculated points
             // Only update points_awarded — never touch the awarded (confirmation) field
             const { error: bonusUpdateError } = await adminClient
               .from('bonus_awards')
               .update({ points_awarded: bonusResult.points_awarded })
               .eq('id', award.id)

             if (bonusUpdateError) {
               errors.push(`Failed to update bonus for award ${award.id}: ${bonusUpdateError.message}`)
             } else {
               bonus_calculated++
             }
           }
         }
       }
       ```

    4. Update the return to include `bonus_calculated`:
       ```typescript
       return {
         fixture_id: fixtureId,
         predictions_scored: scoreRows.length,
         bonus_calculated,
         errors: [],
       }
       ```

    IMPORTANT: Uses admin client (already in scope) — RLS blocks member writes to bonus_awards.
    IMPORTANT: This is idempotent — calling UPDATE on the same bonus_awards row multiple times with the same calculated value is safe. Points are recalculated fresh each time from the prediction_scores data.
    IMPORTANT: Never modify `awarded` field — that's George's confirmation. Only update `points_awarded`.
