    Create `src/actions/admin/fixtures.ts` following the exact pattern from `src/actions/admin/members.ts`:
    - All actions are `'use server'` and start with `requireAdmin()` guard
    - All actions validate input with Zod schemas from validators/admin.ts
    - All actions use `createAdminClient()` for DB writes (bypasses RLS)
    - All actions return `{ success: true, ... }` or `{ error: string }`

    **`addFixture(formData: FormData)`:**
    - Parse and validate with `addFixtureSchema`
    - Look up gameweek UUID by `gameweek_number` (query gameweeks table)
    - Generate a negative `external_id` for manually added fixtures (e.g., `-Date.now()`) to avoid collision with football-data.org IDs which are positive
    - Insert into fixtures table
    - `revalidatePath('/admin/gameweeks')`
    - Return success with new fixture ID

    **`editFixture(formData: FormData)`:**
    - Parse and validate with `editFixtureSchema`
    - Fetch current fixture to compare kickoff_time change
    - If kickoff_time changed, set `is_rescheduled = true`
    - Update fixture with only the provided (non-undefined) fields
    - `revalidatePath('/admin/gameweeks')`
    - Return success

    **`moveFixture(formData: FormData)`:**
    - Parse and validate with `moveFixtureSchema`
    - Look up target gameweek UUID by `target_gameweek_number`
    - If gameweek doesn't exist, return error
    - Fetch current fixture to get current gameweek for notification message
    - Update fixture's `gameweek_id` to target gameweek UUID
    - Create admin_notification type='fixture_moved' with details
    - `revalidatePath('/admin/gameweeks')`
    - Return success

    **`triggerSync()`:**
    - No FormData needed — just requires admin auth
    - Call `syncFixtures()` from `src/lib/fixtures/sync.ts`
    - `revalidatePath('/admin/gameweeks')`
    - Return the sync result object

    - Test addFixture validates input (rejects same team for home and away)
    - Test editFixture updates only provided fields
    - Test moveFixture rejects invalid gameweek number
    - Test triggerSync calls syncFixtures and returns result
    Use the mock Supabase pattern from tests/setup.ts.
