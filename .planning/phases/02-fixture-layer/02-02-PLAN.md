    **`editFixture(formData: FormData)`:**
    - Parse and validate with `editFixtureSchema`
    - Fetch current fixture to compare kickoff_time change

    **CRITICAL — Kickoff guard (Blocker 2, FIX-03 enforcement):**
    After fetching the current fixture, check whether it has already kicked off:
    ```typescript
    const hasKickedOff = new Date() >= new Date(fixture.kickoff_time)

    if (hasKickedOff) {
      // After kickoff, only allow score updates and status changes.
      // Block changes to kickoff_time, home_team_id, away_team_id.
      const isAdminOverride = formData.get('admin_override') === 'true'

      if ((validated.kickoff_time || validated.home_team_id || validated.away_team_id) && !isAdminOverride) {
        return { error: 'Cannot edit fixture details after kick-off. Use admin override if this is intentional.' }
      }
    }
    ```
    This means:
    - Scores (home_score, away_score) and status can ALWAYS be updated (needed for entering results)
    - kick-off time and teams are BLOCKED after kickoff unless `admin_override=true` is explicitly passed
    - The admin override flag is for exceptional cases (e.g., correcting a data entry mistake) — the fixture form UI should present this as a separate "Override" button with extra confirmation

    - If kickoff_time changed (and allowed), set `is_rescheduled = true`
    - Update fixture with only the provided (non-undefined) fields
    - `revalidatePath('/admin/gameweeks')`
    - Return success

    **`moveFixture(formData: FormData)`:**
    - Parse and validate with `moveFixtureSchema`
    - Look up target gameweek UUID by `target_gameweek_number`
    - If gameweek doesn't exist, return error
    - Fetch current fixture to get current gameweek for notification message
    - Update fixture's `gameweek_id` to target gameweek UUID
    - Create admin_notification type='fixture_moved' with details
    - `revalidatePath('/admin/gameweeks')`
    - Return success

    **`triggerSync()`:**
    - No FormData needed — just requires admin auth
    - Call `syncFixtures()` from `src/lib/fixtures/sync.ts`
    - `revalidatePath('/admin/gameweeks')`
    - Return the sync result object
