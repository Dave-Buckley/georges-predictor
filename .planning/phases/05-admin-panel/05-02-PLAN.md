Create `src/actions/admin/bonuses.ts` following the established `requireAdmin()` + `createAdminClient()` + Zod validation + revalidatePath pattern from `src/actions/admin/scoring.ts`.

**Server actions to implement:**

1. `setBonusForGameweek(formData: FormData)` — validates with setBonusSchema, checks for existing member picks (count from bonus_awards where gameweek_id matches), upserts bonus_schedule with confirmed=true/confirmed_at/confirmed_by, creates admin_notification if changing an already-confirmed bonus with existing picks, revalidates /admin/bonuses and /admin/gameweeks.

2. `toggleDoubleBubble(formData: FormData)` — validates with toggleDoubleBubbleSchema, updates gameweeks.double_bubble, revalidates /admin/bonuses and /admin/gameweeks.

3. `confirmBonusAward(formData: FormData)` — validates with confirmBonusAwardSchema, updates bonus_awards.awarded + confirmed_by + confirmed_at, revalidates /admin/bonuses.

4. `bulkConfirmBonusAwards(formData: FormData)` — validates with bulkConfirmBonusSchema, updates all bonus_awards where gameweek_id matches AND awarded IS NULL, sets awarded=true (approve_all) or awarded=false (reject_all), revalidates.

5. `createBonusType(formData: FormData)` — validates with createBonusTypeSchema, inserts bonus_types with is_custom=true, revalidates /admin/bonuses, returns { success: true, id: string }.

All actions: use `'use server'` directive, import requireAdmin from a shared location or inline it (follow existing pattern — each action file has its own requireAdmin), use createAdminClient() for all DB writes, insert audit-relevant admin_notification rows for significant changes (bonus set, bonus changed, Double Bubble toggled).

**Important:** Use Zod v4 pattern: `parsed.error.issues[0]?.message` not `.errors[]`. Use `z.coerce.number()` for any numeric FormData fields. Remember FormData `.get()` returns string|null — use z.string() for UUIDs, z.coerce for numbers, z.enum for string unions. For boolean fields from FormData, parse as `formData.get('enabled') === 'true'` before passing to Zod.

- Test auth rejection, validation failure, and successful operation for each action
- Use valid v4 UUIDs in test constants (version nibble [1-8], variant [89ab])
