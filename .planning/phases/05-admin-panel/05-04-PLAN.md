**Create `src/actions/admin/prizes.ts`:**

1. `confirmPrize(formData: FormData)` — validates with confirmPrizeSchema.
   - requireAdmin() guard
   - UPDATE prize_awards: set status, confirmed_by (auth.userId), confirmed_at (now()), notes (from form)
   - If status='confirmed': INSERT admin_notification (type: 'prize_triggered', title: '{prizeName} awarded to {memberName}')
   - revalidatePath('/admin/prizes')
   - Return { success: true }

2. `createPrize(formData: FormData)` — validates with createPrizeSchema.
   - requireAdmin() guard
   - INSERT additional_prizes with is_custom=true, trigger_config=null
   - revalidatePath('/admin/prizes')
   - Return { success: true, id: string }

3. `checkDatePrizes()` — NOT a form action, a utility function called by the cron route.
   - Get today's date in Europe/London timezone (use the timezone utility from src/lib/fixtures/timezone.ts or manually compute with Intl.DateTimeFormat)
   - Query additional_prizes where trigger_type='date'
   - For each date-based prize, parse trigger_config to get the target date (month + day):
     - Christmas: Dec 25
     - Halloween: Oct 31
     - Easter: variable — compute from trigger_config or use a simple lookup (e.g., store the exact date in trigger_config for the current season)
     - Valentine's: Feb 14
     - Dry January: Jan 31
   - If today matches a prize date:
     - Check if a prize_awards row already exists for this prize in the current period (prevent duplicate triggers)
     - If no existing award: snapshot current standings (query prediction_scores grouped by member_id, sum points_awarded, join with member display_name)
     - INSERT prize_awards with snapshot_data containing the standings snapshot, status='pending'
     - INSERT admin_notification (type: 'prize_triggered', title: '{prizeName} triggered')
   - Return { triggered: string[] } — list of triggered prize names
